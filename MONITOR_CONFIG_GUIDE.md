# V免签监控端配置指南

## 概述
本系统使用V免签监控方案实现个人收款自动化。监控端是一个Android应用，用于监听支付通知并将支付信息发送到服务器。

## 系统架构

```
用户支付 → 监控端Android APP → vmq-notify Edge Function → 自动发货
```

## 配置步骤

### 第一步：服务器端配置

1. **启用V免签支付**
   - 进入管理后台 → 系统设置 → V免签支付设置
   - 勾选"启用V免签支付"
   - 设置通信密钥（用于验证通知）

2. **配置收款码**
   - 进入管理后台 → V免签管理 → 收款码管理
   - 添加微信支付和支付宝的收款码
   - 上传收款码图片到图床，填写图片URL

3. **获取监控端配置地址**
   - 在系统设置页面，找到"监控端配置地址"
   - 点击复制按钮获取完整的回调URL

### 第二步：Android监控端配置

1. **安装监控端APP**
   - 下载V免签监控端APK（开源版本）
   - 在Android手机上安装应用

2. **基础配置**
   - 打开监控端APP
   - 在"回调地址"设置中，粘贴从服务器复制的URL
   - 格式示例：`https://your-supabase-url.supabase.co/functions/v1/vmq-notify`

3. **权限配置**
   - 开启通知监听权限（必需）
   - 开启无障碍服务权限（推荐，用于自动确认支付）
   - 允许后台运行和自启动

4. **测试连接**
   - 在监控端APP中测试连接服务器
   - 确保能够成功发送测试通知

### 第三步：使用流程

1. **用户下单**
   - 用户在前台下单时，系统会自动生成随机金额
   - 用户需要支付完全一致的金额

2. **监控端工作**
   - 监控端监听到支付通知
   - 提取支付金额、时间等信息
   - 发送支付通知到服务器

3. **服务器处理**
   - 服务器根据金额匹配订单
   - 自动分配卡密并更新订单状态
   - 发送邮件通知（如配置）

## 监控端API接口

### 支付通知格式
监控端需要发送以下格式的POST请求到回调URL：

```json
{
  "t": "1617189123",        // 支付时间（时间戳）
  "m": "10.50",             // 支付金额（数字）
  "o": "ORDER123456",       // 订单号（可选）
  "r": "abc123"             // 随机字符串（可选，防重放）
}
```

### 服务器响应

成功响应：
```json
{
  "code": 1,
  "msg": "success"
}
```

错误响应：
```json
{
  "code": -1,
  "msg": "错误信息"
}
```

### 错误代码说明

| 错误代码 | 说明 | 解决方案 |
|---------|------|----------|
| -1 | 未找到匹配的订单 | 检查支付金额是否与订单金额一致 |
| -2 | 没有可用卡密 | 检查商品库存，添加卡密 |
| -3 | 订单已处理 | 订单已发货，无需重复处理 |
| -4 | V免签未启用 | 在系统设置中启用V免签支付 |
| -5 | 参数不完整 | 检查监控端发送的数据格式 |

## 高级配置

- 每台手机独立工作，提高可靠性
- 建议在不同网络环境下部署

### 2. 监控端自动化
- 配置Android任务自动化工具
- 设置定时重启监控端
- 配置异常报警通知

### 3. 安全配置
- 定期更换通信密钥
- 限制监控端IP访问（如有固定IP）
- 监控端日志定期清理

## 故障排除

### 常见问题

1. **监控端无法监听到支付通知**
   - 检查通知权限是否开启
   - 确认微信/支付宝版本支持
   - 检查手机是否开启省电模式

2. **支付后订单未自动发货**
   - 检查监控端网络连接
   - 查看服务器日志是否有收到通知
   - 确认支付金额完全一致

3. **监控端频繁断开连接**
   - 检查手机后台管理设置
   - 将监控端加入后台白名单
   - 关闭手机省电优化

### 调试方法

1. **服务器端调试**
   - 查看Supabase Edge Function日志
   - 检查数据库订单状态
   - 验证通信密钥配置

2. **监控端调试**
   - 查看监控端日志
   - 测试发送通知功能
   - 检查网络连接状态
